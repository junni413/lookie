## 3. Git Convention

### 📌 Commit Type 규칙

| type     | 설명 |
|----------|------|
| feat     | 새로운 기능 추가 |
| fix      | 버그 수정 또는 typo |
| refactor | 리팩토링 |
| design   | CSS 등 사용자 UI 디자인 변경 |
| comment  | 필요한 주석 추가 및 변경 |
| style    | 코드 포맷팅, 세미콜론 누락 등 코드 로직 변경이 없는 경우 |
| test     | 테스트 코드 추가, 수정, 삭제 (비즈니스 로직 변경 없음) |
| chore    | 기타 변경사항 (빌드 스크립트 수정, assets image, 패키지 매니저 등) |
| init     | 프로젝트 초기 생성 |
| rename   | 파일 또는 폴더명 수정 및 이동 |
| remove   | 파일 삭제만 수행한 경우 |
| docs     | 문서 작성 및 수정 |
| build    | 빌드 파일 수정 |
| ci       | CI 설정 파일 수정 |

### 📌 Commit Message 형식
[type]: 작업 내용

**예시**
| type     | 작업 내용 |
|----------|------|
| feat     | 로그인 기능 구현 |
| fix      | 사용자 인증 오류 수정 |
| docs     | README 업데이트|
